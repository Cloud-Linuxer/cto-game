## 개요

신뢰도(Trust)는 AWS 스타트업 타이쿤에서 가장 중요한 자원 중 하나입니다. 투자자와 시장의 신뢰를 나타내며, 투자 유치, 사용자 이탈률, 게임 승패에 직접적인 영향을 미칩니다.

### 설계 원칙

1. **회복 가능성**: 실수를 해도 전략적 플레이로 회복 가능
2. **교육적 피드백**: 신뢰도 변화 원인을 명확히 전달
3. **난이도별 균형**: 모든 난이도에서 승리 가능한 밸런스
4. **장애 극복 보상**: 위기를 극복한 플레이어에게 보너스

## 신뢰도 시스템 구조

### 초기값 (EPIC-04 개선)

| 난이도 | 초기 Trust | 설명 |
|--------|-----------|------|
| EASY | 50 | 여유로운 시작 |
| NORMAL | 50 | 균형잡힌 시작 (40 → 50 상향) |
| HARD | 30 | 타이트한 시작 (25 → 30 상향) |

**변경 사유**:
- 이전 NORMAL 40은 초반 투자 유치(Series A: 25 필요)에 여유가 거의 없어 실수 허용도가 낮았음
- 50으로 상향하여 학습 곡선 개선 및 초반 실수 허용

### 범위 및 임계값

```
0-20:   Danger     (파산 위기, 투자 불가능)
21-40:  Critical   (경고 상태, 투자 제한적)
41-60:  Warning    (주의 필요, 일부 투자 가능)
61-80:  Good       (안정적, 대부분 투자 가능)
81-100: Excellent  (최상, 모든 기회 열림)
```

### 투자 유치 임계값

| 투자 라운드 | EASY | NORMAL | HARD |
|------------|------|--------|------|
| Early Pitch (Turn 2) | 3 | 5 | 8 |
| Series A (Turn 12) | 20 | 25 | 35 |
| Series B (Turn 18) | 35 | 45 | 55 |
| Series C (Turn 23) | 55 | 65 | 75 |
| IPO | 60 | 65 | 85 |

## 획득 및 손실 경로

### 신뢰도 획득 (+)

#### 1. 투자 유치 성공
```
Early Pitch: +5
Series A:    +10
Series B:    +15
Series C:    +15
```

#### 2. 비즈니스 성과
```
마케팅 성공:      +3~+5
제품 출시:        +5~+8
글로벌 진출:      +8~+10
```

#### 3. 회복 메커니즘 (EPIC-04 신규)
```
안정 운영 보너스:  +3 (3턴 연속 용량 80% 이하)
장애 극복 보너스:  +5 (복원력 스택 획득 시)
투명성 보너스:     1.5배 (transparency 태그 선택지)
자연 회복:        +1/턴 (trust < 30일 때)
```

### 신뢰도 손실 (-)

#### 1. 용량 초과 (EPIC-04 개선)

**경고 시스템**:
```
첫 초과:         -50% 페널티 + 경고 메시지
연속 2회 이상:   -100% 전체 페널티
정상화 후 재초과: 다시 -50%부터 시작
```

**페널티 계산**:
```
초과율 10%:  -2 trust (첫 초과 시 -1)
초과율 30%:  -4 trust (첫 초과 시 -2)
초과율 50%:  -6 trust (첫 초과 시 -3)
초과율 100%: -8 trust (첫 초과 시 -4)
```

#### 2. 투자 유치 실패
```
Early Pitch 거절: -3
Series A 거절:    -5
Series B 거절:    -8
Series C 거절:    -10
```

#### 3. 비즈니스 실패
```
마케팅 실패:    -2~-4
제품 실패:      -5~-8
보안 사고:      -15
```

## 회복 메커니즘

### 1. 안정 운영 보너스

**조건**:
- 3턴 연속 용량 80% 이하 유지
- 용량 초과 없음

**보상**:
- Trust +3
- 카운터 리셋 (다시 3턴 후 획득 가능)

**전략적 활용**:
```
Turn 5: 60% 용량 (카운터 1)
Turn 6: 75% 용량 (카운터 2)
Turn 7: 70% 용량 (카운터 3) → Trust +3 획득
Turn 8: 다시 카운터 0부터 시작
```

### 2. 장애 극복 보너스

**조건**:
- 용량 초과 발생 (복원력 스택 획득)

**보상**:
- Trust +5 (장애를 극복한 경험의 가치)
- 용량 초과 페널티와 상쇄되어 순손실 감소

**효과 예시**:
```
Trust 35, 용량 초과 (13% 초과)
→ 페널티: -1 (첫 초과 50% 감소)
→ 장애 극복 보너스: +5
→ 순변화: +4 (35 → 39)
```

### 3. 투명성 보너스

**조건**:
- `transparency` 태그가 있는 선택지
- 용량 경고가 활성화된 상태

**보상**:
- 신뢰도 회복 효과 1.5배

**효과 예시**:
```
선택지: "고객에게 장애 상황 투명하게 공개" (trust: +4)
용량 경고 활성화 → 실제 획득: +6 (4 × 1.5)
```

### 4. 자연 회복

**조건**:
- trust < 30 (위기 상황)

**보상**:
- +1 trust/턴 (시장 안정화)
- trust < 15 (극심한 위기): +2 trust/턴
- 최대 30까지만 자연 회복

**전략적 의미**:
- 신뢰도가 너무 낮아도 완전 포기 상태는 아님
- 시간을 벌면서 회복 기회 확보 가능

### 5. 대안 투자 경로 (EPIC-04 신규)

**브릿지 파이낸싱**:
```
조건:     trust가 정규 투자 필요치의 60% 미만
사용 횟수: 게임당 최대 2회
투자 금액: 정규 투자의 30%
페널티:   지분 5% 추가 희석
```

**정부 지원금**:
```
조건:     trust 상관없음
사용 횟수: 게임당 1회
지원 금액: 2억 원 (고정)
보너스:   trust +3 (정부 인증 효과)
의무 사항: 다음 턴 보고서 제출 필요
```

## 밸런스 시뮬레이션

### 시나리오 1: 공격적 성장 (Aggressive Growth)

```
Turn 1:  50 (시작)
Turn 3:  55 (+5, 마케팅 성공)
Turn 5:  52 (-3, 용량 초과 첫 경고)
Turn 6:  57 (+5, 장애 극복 보너스)
Turn 8:  60 (+3, 안정 운영 보너스)
Turn 12: 70 (+10, Series A 성공)
Turn 18: 85 (+15, Series B 성공)
Turn 25: 88 (IPO 성공, 필요: 65)
```

**결과**: 승리 (공격적이지만 회복 메커니즘 활용)

### 시나리오 2: 보수적 운영 (Conservative Play)

```
Turn 1:  50 (시작)
Turn 3:  53 (+3, 안정 운영 보너스)
Turn 6:  56 (+3, 안정 운영 보너스)
Turn 9:  59 (+3, 안정 운영 보너스)
Turn 12: 69 (+10, Series A 성공)

```

**결과**: 승리 (안정적이고 예측 가능)

### 시나리오 3: 위기 극복 (Crisis Recovery)

```
Turn 1:  50 (시작)
Turn 3:  45 (-5, 실수)
Turn 5:  42 (-3, 용량 초과)
Turn 6:  47 (+5, 장애 극복 보너스)
Turn 8:  50 (+3, 안정 운영 보너스)
Turn 10: 53 (+3, 안정 운영 보너스)
Turn 12: 50 (Series A 실패로 브릿지 파이낸싱 사용)
Turn 14: 55 (+5, 회복)

```

**결과**: 승리 (실수했지만 대안 경로로 회복)

```
Turn 1:  50 (시작)
Turn 3:  42 (-8, 용량 100% 초과)
Turn 5:  34 (-8, 연속 초과)
Turn 7:  26 (-8, 연속 초과)
Turn 9:  18 (-8, 연속 초과)
Turn 10: GAME OVER (trust < 20, 파산 위기)
```

**결과**: 패배 (계속된 인프라 무시로 회복 불가)

## 플레이 스타일별 전략

### 1. 공격적 성장형 (Aggressive Growth)

**특징**:
- 빠른 사용자 확보 우선
- 용량 초과 각오
- 회복 메커니즘 적극 활용

**전략**:
```
1. 초반 마케팅 공격적 투자
2. 용량 초과 발생 시 즉시 인프라 업그레이드
3. 장애 극복 보너스 (+5) 활용
4. 투명성 선택지로 신뢰도 빠르게 회복
```

**권장 난이도**: EASY, NORMAL

### 2. 보수적 운영형 (Conservative)

**특징**:
- 안정적 서비스 최우선
- 용량 80% 이하 유지
- 느리지만 확실한 성장

**전략**:
```
1. 인프라 먼저 업그레이드
2. 안정 운영 보너스 (+3) 3턴마다 획득
3. 용량 초과 절대 방지
4. 투자 유치 임계값 여유있게 확보
```

**권장 난이도**: NORMAL, HARD

### 3. 균형형 (Balanced)

**특징**:
- 성장과 안정의 균형
- 상황에 따라 유연한 대응
- 모든 회복 메커니즘 골고루 활용

**전략**:
```
1. 성장 3턴 → 안정 3턴 반복
2. 안정 운영 보너스 정기적 획득
3. 용량 초과 시 투명성 선택지로 빠른 회복
4. 대안 투자 경로를 비상 수단으로 확보
```

**권장 난이도**: 모든 난이도

### 4. 위기 극복형 (Crisis Management)

**특징**:
- 실수 후 회복에 집중
- 대안 투자 경로 적극 활용
- 자연 회복 + 보너스 조합

**전략**:
```
1. trust < 30 시 자연 회복 (+1~+2/턴) 활용
2. 브릿지 파이낸싱으로 시간 벌기 (2회)
3. 정부 지원금으로 trust +3 확보
4. 장애 극복 보너스 (+5) 최대 활용
```

**권장 난이도**: NORMAL, HARD (위기 시 사용)

## 교육적 메시지

신뢰도 변화 시 플레이어에게 다음과 같은 메시지를 제공합니다:

### 획득 메시지

```
안정 운영 보너스: "3턴 연속 안정적 서비스 운영으로 시장 신뢰도가 상승했습니다. (+3)"
장애 극복 보너스: "장애 극복으로 신뢰도가 회복되었습니다. 위기 대응 능력이 인정받았습니다. (+5)"
투명성 보너스: "투명한 소통이 신뢰 회복을 가속화했습니다. (1.5배 효과)"
자연 회복: "시장이 안정화되면서 신뢰도가 소폭 회복되었습니다. (+1)"
```

### 손실 메시지

```
첫 용량 초과: "⚠️ 서비스 응답 지연 발생! 사용자 불만이 증가했습니다. 다음 턴까지 인프라를 개선하세요."
연속 용량 초과: "🔥 서비스 장애 발생! (연속 N회) 신뢰도가 크게 하락했습니다. 즉시 대응이 필요합니다."
투자 실패: "투자자들이 우려를 표명했습니다. 신뢰도 회복이 필요합니다. (-5)"
```

| 플레이 스타일 | EASY | NORMAL | HARD |
|--------------|------|--------|------|
| 공격적 성장 | ✅ 승리 | ✅ 승리 | ⚠️ 위험 |
| 보수적 운영 | ✅ 승리 | ✅ 승리 | ✅ 승리 |
| 균형형 | ✅ 승리 | ✅ 승리 | ✅ 승리 |
| 위기 극복 | ✅ 승리 | ✅ 승리 | ⚠️ 위험 |

### 회복 가능성 테스트

Trust 손실 후 회복 가능 여부:

| 손실 시나리오 | 회복 턴 수 | 회복 방법 |
|--------------|-----------|----------|
| -10 (단일 실수) | 3-4턴 | 안정 운영 보너스 |
| -20 (연속 실수) | 6-8턴 | 안정 운영 + 투명성 |
| -30 (심각한 실수) | 10-12턴 | 대안 투자 + 모든 보너스 |
